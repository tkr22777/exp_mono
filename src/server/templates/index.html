How to Add a New Experiment

To create a new experiment, follow these guidelines for modular development:

1. Create a New Route & Template

Each experiment should have its own URL path and template:

# In your main app.py or routes.py file
@app.route('/experiments/your-experiment-name')
def your_experiment():
    return render_template('experiments/your_experiment.html')

2. Organize Template Files

Create a structured template directory:

templates/
├── index.html                # Main landing page
├── experiments/              # All experiment templates
│   ├── experiment1.html
│   ├── experiment2.html
│   └── your_experiment.html  # Your new experiment

3. Keep Server Code Modular

Organize your Python server code into modules by experiment:

src/
├── server/
│   ├── app.py                # Main application entry point
│   ├── routes/
│   │   ├── __init__.py
│   │   ├── main_routes.py    # Landing page routes
│   │   ├── experiment1.py    # Routes for experiment 1
│   │   └── your_experiment.py # Routes for your experiment
│   ├── models/               # Database models
│   │   ├── __init__.py
│   │   └── experiment_models.py
│   ├── services/             # Business logic by experiment
│   │   ├── __init__.py
│   │   ├── experiment1_service.py
│   │   └── your_experiment_service.py

5. Static Assets

Organize CSS, JS, and other assets by experiment:

static/
├── css/
│   ├── style.css            # Common styles
│   └── experiments/
│       └── your_experiment.css
├── js/
│   └── experiments/
│       └── your_experiment.js
